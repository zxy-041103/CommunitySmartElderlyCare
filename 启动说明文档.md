# 社区智慧养老监护管理平台 - 启动说明文档

## 项目概述

社区智慧养老监护管理平台是一个面向社区老年人的智能监护和管理系统，提供健康监测、服务预约、紧急求助等功能，支持老人、家属、护工和管理员四种角色。

## 数据库初始化步骤

### 1. 创建数据库

#### 方式一：使用批处理脚本（Windows）
```bash
cd backend
init-database.bat
```

#### 方式二：手动创建（Linux/Mac/Windows）
```bash
mysql -u root -p
```

在MySQL命令行中执行：
```sql
CREATE DATABASE IF NOT EXISTS smart_elderly_care DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
USE smart_elderly_care;
```

### 2. 数据库表结构和初始数据

Spring Boot会在首次启动时自动执行以下脚本：
- `schema.sql` - 创建数据库表结构
- `data.sql` - 插入初始测试数据

## 后端启动步骤

### 1. 配置数据库连接

编辑 `backend/src/main/resources/application.properties`：

```properties
spring.datasource.druid.url=jdbc:mysql://localhost:3306/smart_elderly_care?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai&useSSL=false&allowPublicKeyRetrieval=true
spring.datasource.druid.username=root
spring.datasource.druid.password=your_password
```

### 2. 配置Redis连接

编辑 `backend/src/main/resources/application.properties`：

```properties
spring.redis.host=localhost
spring.redis.port=6379
spring.redis.password=your_redis_password
```

### 3. 启动Redis服务

#### Windows

**方式一：命令行启动**
```bash
cd C:\Redis
redis-server.exe
```

**方式二：后台启动**
```bash
cd C:\Redis
start /B redis-server.exe
```

**方式三：配置为Windows服务（可选）**
```bash
# 以管理员身份运行 PowerShell
redis-server --service-install
redis-server --service-start
```

#### Linux/Mac
```bash
redis-server
```

#### 验证Redis是否运行

**使用redis-cli测试**：
```bash
redis-cli ping
# 预期输出: PONG
```

**使用PowerShell测试**：
```powershell
Test-NetConnection -ComputerName localhost -Port 6379
# 预期输出: TcpTestSucceeded : True
```

**查看后端日志**：
- Redis连接成功：`INFO  Redis连接成功，已启用缓存功能`
- Redis连接失败：`WARN  Redis连接失败，已禁用缓存功能`

**注意**：
- 如果Redis未启动，系统会自动降级到数据库查询，功能正常
- 建议启动Redis以获得更好的性能（缓存加速）

### 4. 启动后端服务

#### 方式一：使用Maven命令
```bash
cd backend
mvn clean install
mvn spring-boot:run
```

#### 方式二：使用IDE
- 使用IntelliJ IDEA或Eclipse导入项目
- 运行 `ElderlyCarePlatformApplication.java` 主类

### 5. 验证后端启动

后端服务启动后，访问以下地址验证：

- 健康检查：http://localhost:8080/api/captcha
- API文档：http://localhost:8080/api/doc.html（需要配置Swagger）

后端默认端口：8080
上下文路径：/api

## 前端启动步骤

### 1. 安装依赖

```bash
cd frontend
npm install
```

### 2. 配置后端API地址

编辑 `frontend/vite.config.js`：

```javascript
server: {
  port: 3000,
  open: true,
  proxy: {
    "/api": {
      target: "http://localhost:8080",
      changeOrigin: true,
    },
  },
}
```

### 3. 启动开发服务器

```bash
npm run dev
```

### 4. 访问前端应用

前端服务启动后，浏览器会自动打开：
http://localhost:3000

如未自动打开，手动访问上述地址。

### 5. 构建生产版本

```bash
npm run build
```

构建产物位于 `frontend/dist` 目录。

## 功能模块

### 老人端
- 首页：健康概览、紧急求助、服务预约
- 健康报告：健康数据查看、趋势分析、异常记录
- 服务预约：预约服务、查看预约记录

### 家属端
- 监护中心：实时查看老人健康数据、服务进度
- 消息中心：接收健康预警、任务通知

### 护工端
- 任务首页：查看待处理求助、待执行服务
- 服务记录：查看服务历史、扫码核销

### 管理员端
- 数据统计Dashboard：健康监测覆盖率、服务核销率等指标
- 用户管理：用户管理、角色管理、家属-老人关联审核
- 系统配置：健康预警阈值、数据备份、语音关键词维护

## 常见问题

### 1. 数据库连接失败
- 检查MySQL服务是否启动
- 验证数据库用户名和密码
- 确认数据库已创建

### 2. Redis连接失败
- 检查Redis服务是否启动
- 验证Redis端口和密码配置

### 3. 前端无法访问后端API
- 检查后端服务是否启动
- 验证跨域配置
- 确认API代理配置

### 4. 验证码无法显示
- 检查浏览器是否支持Canvas
- 清除浏览器缓存
- 检查网络连接
